# MaaS Lease Provider Functionality Research

This folder supports the staged research process for the Lease Providers market vertical.

## Current Status

- Active phase: Phase 6 - functionality matrix complete and awaiting review
- Research markets: South Africa first, followed by Namibia, Botswana, Zambia, Mozambique, Ghana and Nigeria after review checkpoints
- Deliverable: `deliverables/maas_lease_provider_research.html`

## Working Rules

- Do not add providers, products or functionalities without supporting evidence.
- Separate evidence, inference and MaaS interpretation.
- Use `Not found in public sources`, `Unable to confirm`, `Requires validation`, `Inferred from available evidence` or `Provider does not appear to disclose this publicly` where needed.
- Stop at each phase checkpoint for user review before proceeding.

## Data Files

- `data/research_status.json`: phase tracker, scope and review status.
- `data/providers.json`: provider universe.
- `data/products.json`: product catalogue.
- `data/functionalities.json`: provider-product-functionality evidence records.
- `data/capability_dictionary.json`: controlled vocabulary for functionality names.
- `data/evidence_log.json`: source and evidence register.
- `data/validation_questions.json`: gaps, uncertainties and follow-up questions.

## Refreshing the HTML

The HTML deliverable is self-contained and includes the active 22-provider South African universe, the product catalogue, the QA-reviewed functionality matrix, the capability dictionary, MaaS role classifications and Phase 6 matrix summary views. After each research phase, update the data files and mirror the latest dataset into the HTML so it remains shareable as one file.
